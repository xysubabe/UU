# UU 跑腿小程序设计文档

## 项目概述

UU 跑腿是一款微信小程序应用，为用户提供帮买、帮送、帮排队三种跑腿服务。用户可以通过小程序下单、支付、跟踪订单。

**版本：** v1.0
**日期：** 2026-03-19
**作者：** Claude

---

## 1. 技术架构

### 1.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | 微信小程序原生开发 (WXML + WXSS + JS/TS) |
| 后端 | Spring Boot + MyBatis-Plus + Java 17+ |
| 构建工具 | Gradle 8.x |
| 数据库 | MySQL 8.0+ |
| 认证 | 微信小程序登录 + JWT |
| 文件存储 | 本地文件系统（用户头像） |
| 地图 | 无（地址纯手写） |
| 支付 | 微信支付 + Mock接口 |

## 3. 数据库设计

### 表名规范

- 单数形式
- t_ 前缀
- 所有表和字段添加中文注释

### 3.1 用户表 (t_user)

```sql
CREATE TABLE t_user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '用户ID',
    openid VARCHAR(100) UNIQUE NOT NULL COMMENT '微信openid',
    unionid VARCHAR(100) COMMENT '微信unionid',
    nickname VARCHAR(50) COMMENT '昵称',
    avatar_url VARCHAR(500) COMMENT '头像URL（存储相对路径，如：avatar/2024/03/123_1710931200000_abc.jpg）',
    phone VARCHAR(20) COMMENT '手机号',
    status TINYINT DEFAULT 1 COMMENT '状态：1正常 0禁用',
    schema_version INT DEFAULT 1 COMMENT '乐观锁版本号',
    create_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_openid (openid) COMMENT 'openid索引'
) COMMENT='用户表';
```

**头像存储说明：**
- `avatar_url` 字段存储相对路径，格式：`avatar/{年}/{月}/{用户ID}_{时间戳}_{随机码}.{扩展名}`
- 通过API接口访问：`GET /file/avatar/{userId}`
- 文件大小限制：最大 5MB
- 支持格式：jpg、jpeg、png

### 3.2 订单表 (t_order)

```sql
CREATE TABLE t_order (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '订单ID',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    order_code VARCHAR(32) UNIQUE NOT NULL COMMENT '订单编号（格式：UU + 订单ID）',
    service_type TINYINT NOT NULL COMMENT '服务类型：1帮买 2帮送 3帮排队',
    status TINYINT NOT NULL DEFAULT 100 COMMENT '订单状态：100待支付 200待接单 300进行中 600已完成 999已取消',
    title VARCHAR(100) COMMENT '订单标题',
    description VARCHAR(200) COMMENT '订单描述（最大200字符）',
    start_address VARCHAR(200) COMMENT '起点地址（最大200字符）',
    end_address VARCHAR(200) COMMENT '终点地址（最大200字符）',
    end_contact_name VARCHAR(50) COMMENT '终点联系人姓名（最大50字符）',
    end_contact_phone VARCHAR(20) COMMENT '终点联系人电话（11位数字）',
    amount DECIMAL(10,2) COMMENT '订单总金额（用户输入，最多10位整数2位小数，必须大于0）',
    delivery_fee DECIMAL(10,2) COMMENT '跑腿费（与amount相同）',
    payment_id BIGINT COMMENT '支付ID',
    schema_version INT DEFAULT 1 COMMENT '乐观锁版本号',
    create_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_user_id_create (user_id, create_at) COMMENT '用户订单列表查询索引',
    INDEX idx_status (status) COMMENT '状态索引',
    INDEX idx_order_code (order_code) COMMENT '订单编号索引',
    FOREIGN KEY (user_id) REFERENCES t_user(id)
) COMMENT='订单表';
```

**订单状态枚举：**
- 100: 待支付
- 200: 待接单
- 300: 进行中
- 600: 已完成
- 999: 已取消

### 3.3 订单日志表 (t_order_log)

```sql
CREATE TABLE t_order_log (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '日志ID',
    order_id BIGINT NOT NULL COMMENT '订单ID',
    action VARCHAR(50) NOT NULL COMMENT '操作类型',
    description VARCHAR(200) COMMENT '操作描述（最大200字符）',
    schema_version INT DEFAULT 1 COMMENT '乐观锁版本号',
    create_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_order_id (order_id) COMMENT '订单ID索引',
    INDEX idx_order_create (order_id, create_at) COMMENT '按时间查询日志索引',
    FOREIGN KEY (order_id) REFERENCES t_order(id)
) COMMENT='订单日志表';
```

### 3.4 地址表 (t_address)

```sql
CREATE TABLE t_address (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '地址ID',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    contact_name VARCHAR(50) NOT NULL COMMENT '联系人姓名（最大50字符）',
    contact_phone VARCHAR(20) NOT NULL COMMENT '联系人电话（11位数字）',
    province VARCHAR(50) COMMENT '省份',
    city VARCHAR(50) COMMENT '城市',
    district VARCHAR(50) COMMENT '区/县',
    detail_address VARCHAR(200) NOT NULL COMMENT '详细地址（最大200字符）',
    is_default TINYINT DEFAULT 0 COMMENT '是否默认地址：1是 0否',
    schema_version INT DEFAULT 1 COMMENT '乐观锁版本号',
    create_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_user_id (user_id) COMMENT '用户ID索引',
    INDEX idx_user_default (user_id, is_default) COMMENT '查询默认地址索引',
    FOREIGN KEY (user_id) REFERENCES t_user(id)
) COMMENT='地址表';
```

**限制：** 每个用户最多保存3个地址。同一用户有且仅有一个默认地址。

### 3.5 支付表 (t_payment)

```sql
CREATE TABLE t_payment (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '支付ID',
    order_id BIGINT NOT NULL COMMENT '订单ID',
    order_code VARCHAR(32) NOT NULL COMMENT '订单编号',
    amount DECIMAL(10,2) NOT NULL COMMENT '支付金额',
    transaction_id VARCHAR(100) COMMENT '第三方交易ID',
    status TINYINT DEFAULT 0 COMMENT '支付状态：0待支付 1已支付 2已退款',
    schema_version INT DEFAULT 1 COMMENT '乐观锁版本号',
    create_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_order_id (order_id) COMMENT '订单ID索引',
    INDEX idx_order_code (order_code) COMMENT '订单编号索引',
    FOREIGN KEY (order_id) REFERENCES t_order(id)
) COMMENT='支付表';
```

### 4.1 首页模块

**页面结构：**
```
┌─────────────────────────────────────┐
│           UU 跑腿                   │  ← 顶部标题
├─────────────────────────────────────┤
│                                     │
│   ┌──────┐  ┌──────┐  ┌──────┐     │
│   │ 帮买  │  │ 帮送  │  │ 帮排队 │    │  ← 服务入口
│   └──────┘  └──────┘  └──────┘     │
│                                     │
│   ┌───────────────────────────┐     │
│   │ 📦 进行中的订单             │     │  ← 订单卡片（仅当有进行中订单时显示）
│   │ ──────────────────────    │     │
│   │ 帮买 · 奶茶x2             │     │
│   │ 进行中 · 2024-03-19 14:30│     │
│   │ ¥ 25.00               [详情]│     │
│   └───────────────────────────┘     │
│                                     │
├─────────────────────────────────────┤
│  首页  订单  我的                    │  ← 底部Tab（3个）
└─────────────────────────────────────┘
```

**核心功能：**
- 服务类型快速跳转
- 进行中的订单实时展示

**底部Tab说明：**
- 首页：当前页面，显示服务入口和进行中的订单
- 订单：跳转到订单列表页
- 我的：跳转到个人中心页

**底部Tab交互说明：**
- 点击"首页"：刷新首页数据，停留在当前首页
- 点击"订单"：跳转到订单列表页，默认显示"全部"tab
- 点击"我的"：跳转到个人中心页

**底部Tab展示规则：**
- 首页：展示底部Tab（首页、订单、我的）
- 订单列表页：展示底部Tab（首页、订单、我的）
- 个人中心页：展示底部Tab（首页、订单、我的）
- 下单页面（帮买/帮送/帮排队）：不展示底部Tab
- 订单详情页：不展示底部Tab
- 地址列表页：不展示底部Tab
- 新增/编辑地址页：不展示底部Tab

**进行中的订单模块说明：**
- 仅当用户有进行中状态的订单时显示该模块
- 如果有多个进行中订单，进行从下向上的滚动展示
- 每个订单停留3秒，然后自动滚动到下一个进行中订单
- 滚动顺序：按时间顺序从新到旧
- 如果只有一个进行中订单，不进行滚动
- 显示订单基本信息：服务类型、描述、状态、金额、时间
- 点击订单卡片跳转到订单详情页
- 右上角显示"详情"按钮，点击后跳转到订单详情页
- 如果没有进行中订单，该模块不显示

- 已完成、已取消状态的订单不显示任何按钮

## 5. API 接口设计

### 5.1 接口规范

**基础URL：** `https://api.uu.com/v1`

**请求头：**
```
Content-Type: application/json
Authorization: Bearer {jwt_token}
```

**登录校验机制：**

所有业务接口（除认证接口外）都需要进行登录校验，通过JWT Token验证用户身份。

**校验方式：**
- 通过拦截器统一校验请求头中的 `Authorization: Bearer {jwt_token}`
- 校验失败返回 HTTP 401 状态码和错误信息
- 校验通过后，从Token中提取用户ID，通过请求头传递给后续处理

**接口分类：**

| 接口类型 | 示例 | 是否需要登录校验 |
|---------|------|----------------|
| 认证接口 | `POST /auth/wechat-login` | ❌ 不需要 |
| 文件接口 | `GET /file/avatar/{userId}` | ❌ 不需要（公开访问） |
| 业务接口 | 其他所有接口 | ✅ 需要 |

**登录校验失败响应：**
```json
{
  "code": 1002,
  "message": "未授权，请先登录",
  "data": null,
  "timestamp": 1710838800000
}
```

**数据隔离规则：**

所有业务接口必须基于当前登录用户ID进行数据隔离，用户只能访问自己创建的数据：

| 数据类型 | 接口示例 | 隔离规则 |
|---------|---------|---------|
| 订单 | `/order/list`, `/order/detail/{orderId}` | 只能查询/操作当前用户创建的订单 |
| 地址 | `/address/list`, `/address/delete/{addressId}` | 只能查询/操作当前用户的地址 |
| 头像 | `/user/avatar/update` | 只能更新当前用户的头像 |
| 用户信息 | `/user/info` | 只能获取当前用户的信息 |

**数据安全校验：**
- 后端必须验证请求的数据是否属于当前登录用户
- 例如：`GET /order/detail/{orderId}` 需要验证该订单的 `user_id` 是否等于当前登录用户ID
- 若数据不属于当前用户，返回 HTTP 403 禁止访问

**响应格式：**
```json
{
  "code": 0,
  "message": "成功",
  "data": {},
  "timestamp": 1710838800000
}
```

**接口：** `POST /user/avatar/update`

**登录校验：** ✅ 需要

**请求参数：**
```
file: 必填，头像文件（multipart/form-data）
```

**请求头：**
```
Content-Type: multipart/form-data
```

**文件校验规则：**
- 文件大小：最大 5MB
- 文件格式：仅支持 jpg、jpeg、png
- 文件内容：通过魔数验证文件真实类型

**业务规则：**
- 文件命名格式：`{用户ID}_{时间戳}_{随机码}.{扩展名}`
- 存储路径：`uploads/avatar/{年}/{月}/`
- 数据库存储：`avatar_url` 字段存储相对路径（如 `avatar/2024/03/123_1710931200000_abc.jpg`）
- 更新前删除旧头像文件（如果存在且不是默认头像）
- 上传失败时删除临时文件，不更新数据库

**响应数据：**
```json
{
  "code": 0,
  "message": "头像更新成功",
  "data": {
    "userId": "123",
    "avatarUrl": "avatar/2024/03/123_1710931200000_abc.jpg"
  },
  "timestamp": 1710838800000
}
```

**错误码：**
- 4001: 文件过大（超过5MB）
- 4002: 文件格式不支持
- 4003: 文件内容验证失败
- 4004: 文件上传失败
- 4005: 磁盘空间不足

**登录校验：** ❌ 不需要（公开访问）

```

```

**业务规则：**
- 根据 userId 查询数据库获取头像路径
- 如果用户未设置头像，返回默认头像图片
- 读取本地文件并返回文件流
- 设置正确的 Content-Type 响应头（image/jpeg、image/png 等）

**响应数据：**
- 直接返回文件二进制流（Content-Type: image/jpeg 或 image/png）
- 失败时返回 HTTP 404 Not Found

### 5.3 订单接口

#### 5.3.1 创建订单

**接口：** `POST /order/create`

**登录校验：** ✅ 需要

**请求参数：**
```json
{
  "serviceType": 1,
  "startAddressId": 123456,
  "endAddressId": 123457,
  "description": "奶茶x2",
  "amount": 25.00
}
```

**参数验证规则：**
- `serviceType`：必填，服务类型（1帮买 2帮送 3帮排队）
- `startAddressId`：必填，起点地址ID
- `endAddressId`：必填，终点地址ID（帮排队订单时此字段可为空）
- `description`：非必填，订单描述，最大200字符
- `amount`：必填，订单金额，数值格式，大于0，最多10位整数2位小数

**业务规则：**
- `startAddressId`：必填，起点地址ID。后端验证该地址是否属于当前登录用户，从 `t_address` 表查询并保存完整信息到 `t_order` 表的起点地址字段
- `endAddressId`：
  - 帮买订单、帮送订单：必填，终点地址ID。后端验证该地址是否属于当前登录用户，从 `t_address` 表查询并保存完整信息到 `t_order` 表的终点地址字段
  - 帮排队订单：不传或传 `null`。帮排队只有一个排队地点（即起点地址），无需终点地址。`t_order` 表的终点地址相关字段（`end_address`、`end_contact_name`、`end_contact_phone`）都为空
- 保存的完整信息包括：`detail_address` → `startAddress`/`endAddress`，`contact_name` → `endContactName`，`contact_phone` → `endContactPhone`

**帮排队订单请求示例：**
```json
{
  "serviceType": 3,
  "startAddressId": 123456,
  "endAddressId": null,
  "description": "排队办理事项",
  "amount": 10.00
}
```

**响应数据：**

## 8. 业务规则总结

### 8.1 订单规则

- 服务类型：帮买、帮送、帮排队（固定3种，不支持扩展）
- 费用：用户自行输入，必填，数值格式，必须大于0，最多10位整数2位小数
- 状态：待支付 → 待接单 → 进行中 → 已完成 / 已取消
- 取消：待支付、待接单状态可手动取消，进行中及以后不可取消
- 超时：不需要自动取消
- 排序：按状态优先+时间排序（进行中订单优先显示，然后按创建时间倒序）
- 时间显示：混合显示（今天用相对时间如"2小时前"，更早用绝对时间如"2024-03-18 14:30"）
- 无消息推送通知
- 订单编号生成：格式为 `UU + 订单ID`，如 UU12345
- 订单描述（description）为非必填字段，最大200字符

### 8.2 地址规则

- 纯手写方式，不联动地图API
- 每个用户最多保存3个地址
- 同一用户有且仅有一个默认地址
- 设置新的默认地址时，自动取消其他地址的默认状态
- 删除默认地址时，如该用户有其他地址，则不指定新的默认地址
- 联系人姓名最大50字符
- 联系人电话11位数字
- 详细地址最大200字符
- 联系人手机号脱敏显示（如 138****1234）

### 8.2.1 下单页面默认地址联动

- 若用户有设置默认地址，进入下单页面时，起点地址自动填入默认地址的详细地址
  - 帮买订单："购买地址"自动填入默认地址
  - 帮送订单："取件地址"自动填入默认地址
  - 帮排队订单："排队地址"自动填入默认地址
- 若没有默认地址，则显示placeholder提示
- 若用户需要修改自动填充（或者上一次选择的地址）的内容，可以点击地址并从弹出的地址下拉列表中选择新地址或者点击下拉列表末尾的”+ 新增购买或收货地址“ 按钮去新增地址
- 收货地址和送达地址不自动填充，需用户手动选择

### 8.3 支付规则

- 微信支付 + Mock接口（运维专用）
- 无自动超时取消
- Mock接口需要运维角色权限，普通用户无法访问
- 支付失败不影响订单状态，订单保持待支付状态

### 8.4 其他规则

- 用户登录：微信授权登录
- 服务区域：无限制，全国通用
- 评价功能：已规划，MVP阶段不实现
- 小费功能：已规划，MVP阶段不实现

### 8.5 下单表单必填规则

**必填字段：**
- 帮买订单：购买地址、收货地址、跑腿费用
- 帮送订单：取件地址、送达地址、跑腿费用
- 帮排队订单：排队地址、跑腿费用

**非必填字段：**
- 帮买订单：购买清单
- 帮送订单：物品描述
- 帮排队订单：排队事项

**前端验证：**
- 点击"立即下单"按钮时，校验必填字段是否已填写
- 校验跑腿费用是否为有效数值且大于0
- 校验地址字段是否超过200字符限制

**后端验证：**
- 验证必填字段是否为空
- 验证跑腿费用是否为有效数值且大于0
- 验证地址字段长度不超过200字符

- 已取消（999）→ 终态，不可变更
